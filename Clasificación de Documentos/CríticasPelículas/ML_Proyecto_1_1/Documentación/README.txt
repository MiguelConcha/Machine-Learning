== Ejecución del proyecto y archivos ==

> En la carpeta "Documentación" se detalla el PDF del proyecto con los requisitos expresados en clase y en la plataforma.
> En el archivo comprimido "Datos" se encuentran los tres conjuntos de ejemplares (Batch 1, Batch 2 y Batch 3) descritos en el PDF. 
  El archivo Batch_i.arff es el archivo que idealmente se debe utilizar para WEKA en el proyecto de Java entregado como 
  parte de la implementación, 0<=i<=3. 
  Se incluye un archivo "configuraciones.txt" con tal de poder replicar los experimentos con los mismos argumentos para el filtro 
  y los algoritmos.
  Además, incluimos el archivo dividido en dos: el de entrenamiento y el de prueba en una relación 70-30.
  Finalmente, incluimos los archivos que utilizamos originalmente para crear los .arffs y los archivos del web scraper, así
  como un archivo de stopwords sugerido.
> En la carpeta "Implementación" está todo lo referente al proyecto de NetBeans en el cual se logró la vinculación entre WEKA
  y JAVA. Los archivos de ejemplares se colocaron también en este directorio para hacer más sencillo el proceso de prueba de los mismos
  al tener que especificar solamente su nombre y no toda la ruta absoluta a éstos.
  En caso de que el proyecto tenga problemas al importarlo en algún IDE como NetBeans, basta con agregar el weka.jar en las bibliotecas
  del proyecto.
